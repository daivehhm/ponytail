大白话介绍本项目：


## 一、它到底是什么？

**Ponytail = AI编程助手的“极简主义刹车片”**

它的本质是一套规则，强行让AI在写代码前先自我审查：**“这代码非写不可吗？能用现成的吗？能一行写完绝不写两行。”**

它不是一个独立的工具，而是给Cursor、Claude Code、Copilot等AI助手装上的**思维框架**。


## 二、它的核心工作原理（六步决策阶梯）

每次AI准备动手写代码时，必须依次问自己这6个问题：

| 步骤 | 问题 | 结论 |
|:---:|:---|:---|
| 1 | 这功能现阶段真的需要吗？ | 不需要 → 直接跳过 |
| 2 | 编程语言自带的标准库能干吗？ | 能 → 直接用，不写新代码 |
| 3 | 操作系统有现成的接口吗？ | 有 → 直接调，不写新代码 |
| 4 | 已装的第三方库能搞定吗？ | 能 → 复用，不写新代码 |
| 5 | 能不能用一行代码写完？ | 能 → 只写这一行 |
| 6 | 以上都不行 | 写“刚好能用”的最少代码 |


## 三、它能帮你做什么（主要功能）

| 功能 | 怎么触发 | 它帮你干啥 |
|:---|:---|:---|
| **自动防过度设计** | 默认开启 | 写代码前自动走上面的六步决策，砍掉不必要的抽象层、设计模式、冗余封装 |
| **审查本次改动** | `/ponytail-review` | 你改完代码后运行，它专门挑出画蛇添足的部分，建议你删掉 |
| **整理技术债务** | `/ponytail-debt` | 把代码里注释标记为 `ponytail:` 的临时凑合方案整理成一张待办清单 |
| **调节管束力度** | `/ponytail ultra/full/lite/off` | 从“往死里省”到“彻底闭嘴”四档可调 |


## 四、什么场景该用它，什么场景不该用

### ✅ 最佳使用场景（推荐开启）

| 场景 | 推荐模式 | 预期效果 |
|:---|:---|:---|
| **从零搭新项目** | `full` | 砍掉过度分层，代码量减少60-80% |
| **写一次性数据处理脚本** | `ultra` | 只写核心逻辑，代码量减少90%+ |
| **写云函数/Serverless** | `ultra` | 减少依赖体积，冷启动从800ms降到200ms |
| **写工具函数/算法片段** | `ultra` | 拒绝类封装，只用纯函数 |

### ❌ 禁用场景（务必先关掉）

| 场景 | 操作 | 原因 |
|:---|:---|:---|
| **修线上紧急Bug** | 先 `/ponytail off` | 别让AI为了简洁而删掉调试日志或异常捕获 |
| **大范围重构遗留代码** | 全程关闭 | 极简主义不适合处理复杂的历史包袱 |
| **多轮长对话（超10轮）** | 关掉或切 `lite` | 规则重复注入会导致Token成本反超收益 |


## 五、具体怎么装、怎么用

### 安装方式

| 你用的工具 | 怎么装 |
|:---|:---|
| **Claude Code / Codex** | 执行插件市场安装命令（见项目README） |
| **Cursor / Windsurf / Cline** | 不需要安装，直接把GitHub仓库里对应文件夹（如 `.cursor/rules/`）的规则文件复制到你的项目根目录即可 |

### 日常使用三板斧

```
# 1. 调整强度（默认是full）
/ponytail ultra      # 最狠模式，能删的全删
/ponytail lite       # 宽松模式，少管一点
/ponytail off        # 彻底关闭，让AI自由发挥

# 2. 审查本次改动有没有过度设计
/ponytail-review

# 3. 查看之前留下的极简“债”
/ponytail-debt
```


## 六、一句话总结

> **Ponytail就是给AI装了个“反对过度设计”的闹钟——写代码前先逼自己三思，能用现成的绝不新写，能写一行绝不写两行。新项目、小脚本、云函数用它能省70%代码和Token钱；修紧急Bug、改老代码时记得先关掉。**
